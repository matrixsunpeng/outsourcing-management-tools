# 外包预算滚动预测 — 运行与计算逻辑

## 目录结构

```
外包预算滚动预测/
├── run_budget_update.bat          # Windows 一键启动
├── run_budget_update.py           # 编排层（交互菜单 + 流程调度）
├── merge_data.py                  # 数据合并（下载文件 → 目标 Excel）
├── generate_dashboard.py          # 仪表盘生成（读 Excel → 计算 → HTML + 导出 Excel）
├── dashboard.html                 # 生成的交互式仪表盘
├── settlement/
│   └── settlement_downloader.py   # 计提结算下载（Playwright 自动化）
├── contract/
│   └── contract_downloader.py     # 外包合同下载（Playwright 自动化）
├── 26年全面预算.xlsx               # 年度预算（BU + 费用 + HC）
├── 费用结算单计提与结算金额查询.xlsx  # 计提结算实际成本
├── 技术合作订单查询报表.xlsx        # 订单明细（用于在岗人数和续签预测）
├── 26年继续投入.xlsx               # 继续投入/实习生转外包计划
├── 资源池列表.xls                  # 资源池预算和 HC（实际为 CSV）
├── 滚动预测update.xlsx             # Excel 导出模板（仅复制样式用）
└── 滚动预测_YYYYMMDD.xlsx          # 输出：按锚点时间切割的滚动预测 Excel
```

## 一、入口与编排（run_budget_update.py）

### 启动方式

```
run_budget_update.bat  →  python run_budget_update.py
```

bat 文件设置 UTF-8 编码后调用 Python 脚本，异常时 `pause` 防止窗口闪退。

### 交互菜单

| 选项 | 流程 |
|------|------|
| 1 | 下载计提结算 → 合并 → 生成仪表盘 |
| 2 | 下载外包合同 → 合并 → 生成仪表盘 |
| 3 | 两个都下载 → 合并 → 生成仪表盘 |
| 4 | 仅合并 → 生成仪表盘（不下载，用已有下载文件） |
| 5 | 仅生成仪表盘（用现有 Excel 数据） |

### 执行流程

```
main()
  ├─ 输入用户名/密码
  ├─ 用户选择 1-5
  ├─ [可选] run_download() → 子进程调用 settlement_downloader.py / contract_downloader.py
  │    └─ 下载前 clear_downloads() 清空旧文件
  ├─ [可选] run_merge() → 子进程调用 merge_data.py
  └─ run_dashboard()
       ├─ 子进程调用 generate_dashboard.py → 生成 dashboard.html + 滚动预测 Excel
       ├─ os.startfile() 打开浏览器
       └─ 线程启动导出 HTTP 服务（端口 8765）
```

## 二、数据合并（merge_data.py）

### 两个目标文件

| 目标 | 源文件 | 主键 |
|------|--------|------|
| `费用结算单计提与结算金额查询.xlsx` | `settlement/downloads/计提结算_*.xlsx` | 技术合作订单编号 + 月份 + 单据类型 |
| `技术合作订单查询报表.xlsx` | `contract/downloads/外包合同_*.xlsx` | 订单编号 + 身份证号 |

### 计提结算合并（追加模式）

```
1. 读取目标文件已有记录 → 构建主键集合
2. 遍历下载文件，找出主键不在目标中的新记录
3. 用 openpyxl 在目标文件末尾追加新行（不动已有数据）
```

### 外包合同合并（同步模式）

```
1. 解析 --contract-start / --contract-end 参数（YYYYMM）
2. 读取目标文件已有记录 → 构建主键集合
3. 读取所有下载文件 → 合并去重
4. 将原记录分为两类：
   - 范围外记录（订单月份不在 start_month~end_month）→ 保留
   - 范围内记录 → 用下载数据整体替换
5. 下载数据中主键在原清单不存在的 → 新增追加
6. 范围内主键在原清单存在但下载无 → 删除
7. 用 openpyxl 重写整个目标文件（范围外保留 + 范围内同步 + 新增）
```

当没有 `contract-start/contract-end` 参数时，退化为纯追加模式（同计提结算）。

### 安全机制

- 写入前自动备份（`.bak`）
- OLE2 格式文件自动转换为 XLSX
- 文件被占用时（PermissionError）报错退出，不损坏原文件
- 写入失败时从 `.bak` 恢复

## 三、仪表盘生成（generate_dashboard.py）——核心计算逻辑

### 数据读取阶段

```
1. 26年全面预算.xlsx        → budget[BU] = {年度费用(千元), 年化HC(人)}
2. 费用结算单计提与结算金额查询.xlsx
   → 按 BU 汇总每月 当月计提
   → ac_bu[BU][1..12] = 实际费用(千元)
3. 技术合作订单查询报表.xlsx
   → 过滤 单据状态 == "审批流程结束" → 计算每月在岗人数
   → ah_bu[BU][1..12] = 实际HC
4. 26年继续投入.xlsx
   → ch_bu[BU][1..12] = {intern, cont} 人数
   → cc_bu[BU][1..12] = {intern, cont} 费用(千元)
5. 资源池列表.xls → 过滤有订单数据的池，按 BU 分组
6. 从订单报表提取 staff_list（全部记录，不限单据状态）
   每条记录: {bu, pool, ws(工作开始), we(工作结束), rate(费率), mp[1..12](月单价)}
```

### 续签预测计算（calcRenewal）

**口径：锚点当天在岗人员，按相同人员逐月核算费用**

```
1. 人员筛选：工作开始时间 ≤ 锚点日期 ≤ 工作结束时间
2. 人数：筛选后在岗人数即为续签人数（所有锚点后的月份人数相同）
3. 费用（从锚点月到 12 月）：

   每月费用 = Σ person( 月单价 × 费率% ÷ 100 ÷ 1.06 ) ÷ 1000

   - 月单价：从技术合作订单的第 4~15 列（一月标准单价 ~ 十二月标准单价）取值
   - 费率：订单的费率字段（0-100）
   - ÷100：费率从百分比转为小数
   - ÷1.06：去除 6% 增值税（含税价 → 不含税价）
   - ÷1000：元 → 千元
   - 结果向上取整（math.ceil）
```

### BU 卡片计算（calcBU）

```
对每个 BU，逐月计算：

1. 锚点月之前（m < anchMonth）：已发生数据
   费用 = 实际计提费用 + 继续投入费用(实习生 + 外包)
   HC   = 实际在岗人数 + 继续投入人数(实习生 + 外包)

2. 锚点月及之后（m ≥ anchMonth）：预测数据
   费用 = 续签预测费用 + 继续投入费用(实习生 + 外包)
   HC   = 续签人数 + 继续投入人数(实习生 + 外包)

3. 汇总：
   预测年度费用 = Σ(1..12) 每月费用
   预测年化 HC   = ceil(Σ(1..12) 每月HC / 12)

4. 进度百分比：
   费用进度 = 预测年度费用 / 年度预算费用 × 100%
   HC 进度  = 预测年化HC / 年化预算HC × 100%
```

### BU 数据来源简化表

| 阶段 | 费用来源 | HC 来源 |
|------|---------|---------|
| 1 月至锚点月-1 | 实际计提 + 继续投入(费) | 实际在岗 + 继续投入(人) |
| 锚点月至 12 月 | 续签预测(费) + 继续投入(费) | 续签(人) + 继续投入(人) |

### 资源池明细计算

```
对每个资源池：
  实际发生年费 = Σ(锚点月之前) 该池实际计提费用
  续签年费     = Σ(锚点月至12月) 该池续签预测费用
  预测年度费用 = 实际发生年费 + 续签年费
```

### 仪表盘输出

- `dashboard.html`：内嵌全量 JSON 数据，纯前端 JS 渲染（含 BU 卡片、三页签明细表、资源池可展开表）
- `滚动预测_YYYYMMDD.xlsx`：按 `BU × 科目 × 月份` 的二维表，锚点前填实际值、锚点后填预测值

## 四、下载器（settlement / contract downloader）

两个下载器共享相同骨架模式：

```
1. 以 headless 模式启动 Playwright Chromium
2. 登录 IMS (https://ims.asiainfo.com/AIOMS/Jsp/main.jsp)
3. 左侧菜单导航（双击父菜单"外包数据查询"展开 → 单击子菜单）
4. 填写查询条件（SBU / 时间范围 / 服务类型 / 单据状态等）
5. 点击查询 → 等待加载 → 点击导出
6. 文件保存到 downloads/ 目录，命名含 SBU + 时间 + 时间戳
```

| 下载器 | 页面框架 | 特殊处理 |
|--------|---------|---------|
| 计提结算 | MiniUI（iframe） | 通过 label 文本定位 combobox，月份范围直接输入 |
| 外包合同 | MiniUI（新标签页） | 技术合作种类按 UI 索引选，单据状态多选，导出按钮为"导出合同" |

## 五、生成 Excel 导出公式（exportExcel / export_rolling_forecast）

HTML 前端通过 SheetJS 生成 Excel，Python 端通过 openpyxl 生成。两者输出格式一致：

```
表头: BU | 科目 | 单位 | 202601 | 202602 | ... | 202612

每个 BU 输出 6 行：
  - 实际发生人数    （锚点月前=实际HC，    锚点月后=空）
  - 实际发生费用    （锚点月前=实际费用，  锚点月后=空）
  - 预测续签人数    （锚点月前=空，        锚点月后=续签HC）
  - 预测续签费用    （锚点月前=空，        锚点月后=续签费用）
  - 继续投入人数    （全年=继续投入HC合计）
  - 继续投入费用    （全年=继续投入费用合计）
```

BU 顺序固定：CMB, CSC, CMD, CUC, CTC, AID, ESU, AIO, RIC, CSU。
