# 续签自动下单工具 - 使用指南

## 📋 功能概述

这是一套完整的外包续签自动下单工具，可自动执行以下流程：

1. **下载数据** - 从企业系统下载续签查询报表
2. **生成清单** - 筛选符合条件的待办记录，生成待办清单（含处理状态列）
3. **自动下单** - 登录网站，在"外包合同"页面逐条填表、计算成本、提交审批
4. **结果追踪** - 处理完成后回写每条记录的处理状态（成功/失败/异常原因）

## 🚀 快速开始

### 1. 环境配置

#### 安装依赖
```bash
pip install -r requirements.txt
```

#### 确认账号配置
编辑 `下载外包续签查询/config.py`，确保账号和密码正确：
```python
USERNAME = ""
PASSWORD = ""
DOWNLOAD_DIR = "./downloads"
```

### 2. 运行方式

有三种使用模式：

#### 模式1：完整流程（推荐首次使用）
```bash
python main.py --headless
```
- 自动下载最近一个月的数据
- 生成待办清单
- 自动登录并逐条下单
- 保存处理结果

自定义日期范围和 SBU：
```bash
python main.py \
  --start "2026年3月1日" \
  --end "2026年3月31日" \
  --sbu "财务部,人力资源部" \
  --headless
```

#### 模式2：仅下载并生成清单
如果想先查看待办清单，再决定是否下单：
```bash
python main.py --download-only --headless
```

输出文件：`待办订单清单_YYYYMMDD_HHMMSS.xlsx`

此后可手动修改清单，再运行模式3进行下单。

#### 模式3：仅处理待办清单
使用已生成的清单文件进行下单：
```bash
python main.py --process-only 待办订单清单_20260331_150000.xlsx --headless
```

输出文件：`待办订单清单_已处理_YYYYMMDD_HHMMSS.xlsx`

### 3. 快速启动（交互式）
```bash
python quick_start.py
```
按提示选择模式和输入参数。

## 📊 输出文件说明

### 待办清单 (待办订单清单_*.xlsx)

**生成时机**：下载完成后自动生成

**内容**：筛选条件
- ✓ "待续签申请单" 不为空
- ✓ "单据状态" = "审批流程结束"
- ✓ "离职时间" 为空

**列名**：
| 列 | 说明 | 备注 |
|----|------|------|
| 待续签申请单 | 申请单编号 | 示例：202603012818Y79-0007 |
| 技术合作商名称 | 供应商名称 | 例：上海XX技术有限公司 |
| 工作地点 | 工作地点代码 | 来自原始报表 |
| 处理状态 | 初始为空 | 等待处理 |

**用途**：
- 审查待办项
- 修改工作地点（如需）
- 在模式3中作为下单输入

### 处理结果文件 (待办订单清单_已处理_*.xlsx)

**生成时机**：下单流程完成后自动生成

**特点**：与待办清单相同结构，但 "处理状态" 列已填充

**处理状态值示例**：
| 状态 | 含义 | 处理建议 |
|------|------|--------|
| ✅ 已提交审批 | 成功提交 | 无需处理 |
| ❌ 错误：人员信息为空 | 系统无该申请单 | 检查申请单号或重新查询 |
| ❌ 错误：订单金额为空 | 成本计算异常 | 登录网站手动检查 |
| ❌ 异常：[具体错误] | 其他异常 | 查看日志和截图调试 |

### 日志文件 (logs/renewal_order_*.log)

**内容**：完整的执行日志，包含每条记录的处理过程

**用途**：调试失败的记录

## ⚙️ 高级用法

### 1. 指定输出目录
```bash
python main.py --output-dir ./results/ --headless
```

### 2. 单个 SBU 处理
```bash
python main.py --sbu "财务部" --headless
```

### 3. 浏览器默认可见
所有命令都支持 `--headless` 参数，用于查看网站操作过程。

### 4. 命令行批处理
```bash
# 依次处理多个期间
for month in 1 2 3; do
  start="2026年${month}月1日"
  end="2026年${month}月30日"
  python main.py --start "$start" --end "$end" --download-only
done
```

## 🐛 问题排查

### 问题：待办清单为空
**原因**：下载的报表中无符合条件的记录

**解决**：
1. 检查日期范围是否正确
2. 登录网站手动验证是否有待续签的申请单
3. 查看日志文件了解过滤情况

### 问题：下单失败 - "人员信息为空"
**原因**：系统中没有该申请单或申请单号输入有误

**解决**：
1. 检查 "待续签申请单" 列的内容
2. 登录网站手动搜索验证申请单是否存在
3. 如申请单正确，可能系统数据延迟，稍后重试

### 问题：下单失败 - "订单金额为空"
**原因**：成本计算异常，可能页面加载不完全

**解决**：
1. 增加超时等待时间（修改 `renewal_order_submitter.py` 中的 `wait_for_timeout`）
2. 查看日志文件和任务监视器截图
3. 手动登录网站检查该记录是否能计算成本

### 问题：浏览器页面找不到元素
**原因**：页面 CSS 选择器可能变更

**解决**：
1. 运行 `--headless` 看实际页面
2. 在浏览器开发者工具中查找正确的元素选择器
3. 修改 `renewal_order_submitter.py` 中 SELECTORS 字典

### 问题：登录失败
**原因**：账号、密码或网站登录页面变更

**解决**：
1. 确认 `config.py` 中的账号密码正确
2. 手动登录验证账户是否可用
3. 检查网络连接

## 📝 数据流向

```
下载报表 (Excel)
    ↓
筛选符合条件的记录
    ↓
待办清单 (待办订单清单_*.xlsx)
    ├─ [可选修改]
    ↓
自动登录 + 逐条下单
    ├─ 选择商名
    ├─ 搜索申请单
    ├─ 填工作地点
    ├─ 计算成本
    ├─ 验证金额
    └─ 提交审批
    ↓
处理结果 (待办订单清单_已处理_*.xlsx)
    ├─ 成功: ✅ 已提交审批
    ├─ 失败: ❌ 错误原因
    └─ 异常: ❌ 异常信息
```

## 📞 技术支持

### 查看完整日志
```bash
cat logs/renewal_order_YYYYMMDD_HHMMSS.log | grep ERROR
```

### 查看特定记录的处理过程
```bash
grep "202603012818Y79-0007" logs/renewal_order_*.log
```

### 重新处理失败的记录
1. 从处理结果文件中筛选失败的记录
2. 修改状态列清空失败标记（或创建新清单）
3. 使用模式3重新处理

## 🎯 最佳实践

1. **首次使用**：
   - 先用 `--download-only` 查看清单
   - 确认清单无误后再执行完整流程

2. **大批量处理**：
   - 分期间下载（按月或按季）
   - 每期间分别下单，便于问题追踪

3. **失败恢复**：
   - 保留所有生成的清单和结果文件
   - 定期查看处理结果，及时处理失败项

4. **调试**：
   - 始终使用 `--headless` 查看实际操作
   - 出问题时截图保存，便于后续分析
