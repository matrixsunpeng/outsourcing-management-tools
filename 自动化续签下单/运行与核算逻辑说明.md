# 自动化续签下单 — 运行与核算逻辑说明

## 一、功能概述

自动完成外包续签的三个环节：**下载报表 → 生成待办清单 → 逐条下单提交审批**。内部集成了"离岗不续签人员自动删除"能力。

目标系统：`https://ims.asiainfo.com/AIOMS/Jsp/main.jsp`（MiniUI 框架）。

---

## 二、运行模式

由 `quick_start.py` 或 `main.py` 启动，共三种运行模式：

| 模式 | 入口 | 说明 |
|------|------|------|
| 完整流程 | `main.py` 无特殊参数 | 下载报表 → 生成清单 → 逐条下单 |
| 仅下载 + 清单 | `main.py --download-only` | 只下载并生成清单，不下单 |
| 仅处理清单 | `main.py --process-only <文件>` | 跳过下载，直接下单已有的待办清单 |

### 关键参数

| 参数 | 说明 | 格式示例 |
|------|------|----------|
| `--sbu` | SBU 名称，逗号分隔，不填=全部 | `CMB,CUC` |
| `--start` / `--end` | 续签查询日期范围 | `2026年3月31日` |
| `--audit-date` | 稽核时间，用于筛选离岗人员 | `2026-03-31`，不填默认=end |
| `--download-only` | 仅下载，不下单 | — |
| `--process-only` | 仅处理已有清单 | — |
| `--skip-delete` | 跳过离岗人员删除 | — |
| `--resume` | 断点续跑（跳过已有反馈的行） | 仅 `--process-only` 模式 |

---

## 三、多 SBU 运行机制

当指定多个 SBU（如 `CMB,CUC`）时，每个 SBU 分别下载三份报表：

```
SBU=CMB:
  ├─ 续签查询报表（CMB）     —— 独立的 Excel 文件
  ├─ 申请单报表（CMB）       —— 独立的 Excel 文件
  └─ 人员变更报表（CMB）     —— 独立的 Excel 文件

SBU=CUC:
  ├─ 续签查询报表（CUC）
  ├─ 申请单报表（CUC）
  └─ 人员变更报表（CUC）
```

**下载完成后，所有同类型报表合并处理：**

- **待办订单清单** = CMB 续签查询 + CUC 续签查询 → 合并去重（按"合作申请单编号"）
- **离岗不续签清单** = CMB 人员变更 + CUC 人员变更 → 合并去重（按"身份证号"）
- **申请单过滤集** = CMB 申请单 + CUC 申请单 → 合并（所有"合作申请单编号"的并集）

每一轮下载都从**干净的页面**开始（上一轮的标签页会被关闭后重新导航），避免表单条件串扰。

## 四、完整流程分步说明

### 阶段 1：下载续签查询报表

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1.1 | 登录 IMS | 自动填写用户名/密码 → 点击登录 |
| 1.2 | 导航 | 左侧树：双击"外包数据查询" → 单击"续签查询" |
| 1.3 | 查询条件 | SBU × 1、申请时间范围 = start~end |
| 1.4 | 导出 | 点击"导出待续签人员" |
| 1.5 | 循环 | 如果多个 SBU，重复 1.3~1.4 |

**输出**：每个 SBU 一个 `续签查询_*.xlsx` 文件。

### 阶段 2：下载申请单报表

| 步骤 | 操作 | 说明 |
|------|------|------|
| 2.1 | 导航 | 左侧树：双击"外包数据查询" → 单击"申请单"（新标签页） |
| 2.2 | 查询条件 | 填入 4 个条件（见下方） |
| 2.3 | 导出 | 点击"查询" → 等待结果 → 点击"导出Excel" |
| 2.4 | 循环 | 每个 SBU 重复 2.1~2.3（上一轮标签页关闭后重开） |

**申请单查询条件（4 个）**：

| 条件 | 控件 ID | 值 |
|------|---------|----|
| 申请时间 | `p_apply_begin_date` / `p_apply_end_date` | 结束时间 = end，开始时间 = end 前 30 天 |
| 技术合作种类 | `p_coop_type` | "技术合作-\|\|"（buttonedit 点击选） |
| 单据状态 | `p_app_state` | "审批流程结束"（buttonedit 点击选） |
| SBU | `p_sbu_id` | 当前 SBU 名称（文本匹配） |

**输出**：每个 SBU 一个 `申请单_<SBU>_*.xlsx` 文件。

### 阶段 3：下载人员变更报表

| 步骤 | 操作 | 说明 |
|------|------|------|
| 3.1 | 登录 + 导航 | 独立浏览器会话（外部模块 `下载人员变更`） |
| 3.2 | 查询条件 | SBU × 1、申请时间 = end 前 2 个月 ~ end |
| 3.3 | 导出 | 标准导出流程 |

**输出**：每个 SBU 一个人员变更 Excel 文件。

#### 四个筛选条件详解

**条件 1 — 待续签申请单非空**：续签查询报表中的"待续签申请单"列不能为空。这列存储的是与该人员关联的 IMS 合作申请单编号。

**条件 2 — 单据状态 = "审批流程结束"**：只处理已完成审批流程的申请单。处于审批中、草稿、驳回等状态的单据不应续签。

**条件 3 — 离职时间为空**：如果续签查询报表中该人员的"离职时间"列已有值，说明该人员正在办理离职流程，不应选中续签。

**条件 4 — 申请单号在有效集合内**：续签查询报表中的"待续签申请单"编号必须出现在申请单报表的"合作申请单编号"中。这是为了排除那些虽然在续签查询中出现、但实际在 IMS 申请单系统中不存在的编号（可能是数据残留）。

#### 离岗不续签筛选逻辑

**时间窗口**：`[稽核时间 - 20 天, 稽核时间]`

- **上限**（稽核时间）：预计离岗时间晚于稽核时间的，说明离岗还在较远的未来，本次续签先不处理
- **下限**（稽核时间前 20 天）：预计离岗时间早于 20 天前的，说明离岗时间太久远，可能已经是过期数据
- **20 天窗口的逻辑**：筛选出"即将在稽核时间前离岗"的人员——这些人本次合同周期内就会离开，不需要续签

**单据状态**：只保留"审批流程中"或"审批流程结束"的单据。排除草稿、驳回等无效状态。

**去重规则**：同一身份证号如果有多条人员变更记录，只保留"预计离岗时间最迟"的那条。确保取到最新的离岗计划。

### 阶段 5：逐条下单

对 `待办订单清单` 中的每一行（除已有反馈的行外），执行：

```
┌──────────────────────────────────────────────────────┐
│              单条订单处理流程                           │
├──────────────────────────────────────────────────────┤
│                                                      │
│  0. 清空表单 + 点击"新增"按钮                           │
│       ↓                                              │
│  1. 选择技术合作商                                     │
│       ↓                                              │
│  2. 搜索申请单（输入编号 → 系统加载人员数据）              │
│       ↓                                              │
│  2.5 填写签约方（如有）                                │
│       ↓                                              │
│  3. 填充工作地点（使用待办清单中的值）                    │
│       ↓                                              │
│  4. 删除不续签人员（按身份证号匹配，可选）                 │
│       ↓                                              │
│  5. 计算成本（轮询等待金额回填，最多 30 秒）               │
│       ├─ 弹窗出现 → "合同未签署：<弹窗内容>"              │
│       ├─ 金额为空/0 → "合同未签署：金额为空或为0"          │
│       └─ 成功 → 继续                                   │
│       ↓                                              │
│  6. 验证金额                                          │
│       ↓                                              │
│  7. 提交审批                                          │
│       ├─ 成功 → "审批提交成功：<申请单号>"               │
│       ├─ 金额未保存 → "合同未签署：...请手动检查"         │
│       └─ 其他失败 → 具体错误信息                        │
│       ↓                                              │
│  8. 返回列表页（无论成功失败）                           │
│                                                      │
│  结果 → 回写到待办清单的「反馈」列                       │
│                                                      │
└──────────────────────────────────────────────────────┘
```

#### 关键步骤说明

**步骤 4 — 人员删除**：在申请单人员表格中，按"身份证号"匹配 `离岗不续签清单` 中的人员，勾选 → 删除。如果 `--skip-delete` 或未提供离岗清单，跳过此步骤。

**步骤 5 — 计算成本**：点击"计算成本"按钮后，分两阶段轮询：
- 阶段 A：等待 `changeOrderAmount` 出现（前端已响应）
- 阶段 B：等待 `p_order_amount` 出现（服务端异步计算完成）
- 超时 30 秒 → 失败
- 出现弹窗 → 判定为"合同未签署"，记录弹窗内容

**步骤 7 — 提交审批**：点击"保存并提交审批"，等待提交结果弹窗。弹窗文本含"成功"即视为提交成功。

## 五、断点续跑

当 `--resume` 开启时，逐条处理会跳过「反馈」列非空的行。

适用场景：
- 下单中断（网络断开、浏览器崩溃等）
- 待办清单中前面部分已处理完成

使用方式：
```bash
# 用已处理的清单文件继续跑，跳过已完成的
python main.py --process-only 待办订单清单_已处理_xxx.xlsx \
               --not-renewing-file 离岗不续签清单_xxx.xlsx \
               --resume
```

## 七、日期计算规则

### 申请单的申请时间范围

```
申请结束时间 = 续签查询结束时间（end）
申请开始时间 = 续签查询结束时间 - 30 天
```

**逻辑**：申请单的审批流程通常需要一定周期，往前取 30 天覆盖从申请到审批完成的整个时间段。

### 人员变更的申请时间范围

```
申请结束时间 = 续签查询结束时间（end）
申请开始时间 = 续签查询结束时间 - 2 个月（月末边界安全处理）
```

**逻辑**：人员变更（如离职申请）的提交时间可能早于合同结束时间较多。往前取 2 个月覆盖足够的回溯范围，确保不遗漏即将离岗的人员。月末边界处理：如 3 月 31 日往前推 2 个月 → 1 月 31 日；如 3 月 30 日 → 1 月 30 日。

### 稽核时间与离岗筛选时间窗口

```
稽核时间 = 用户指定 or 续签查询结束时间（end）
```

用于筛选离岗不续签的窗口：

```
实际离岗时间下限 = 稽核时间 - 20 天（不含更早的过期数据）
实际离岗时间上限 = 稽核时间（不含更晚的未来离岗）
```

## 八、数据去重规则

| 清单 | 去重键 | 保留策略 |
|------|--------|----------|
| 待办订单清单 | `合作申请单编号` | 合并多 SBU 后按此键去重 |
| 离岗不续签清单 | `身份证号` | 同人保留离岗时间最迟的记录 |
| 申请单过滤集 | `合作申请单编号` | 多 SBU 取并集（自动 set 去重） |

---

## 九、异常处理

| 场景 | 处理方式 |
|------|----------|
| 申请单下载失败 | 不中断主流程，不过滤申请单号（所有续签记录都保留） |
| 人员变更下载失败 | 不中断主流程，不下发离岗清单（下单时跳过人员删除） |
| 人员变更模块不存在 | 同上，可独立运行 |
| 单条下单失败（合同未签署） | 记录错误原因到「反馈」列，继续下一条 |
| 单条下单失败（弹窗异常） | 记录错误信息，继续下一条 |
| 浏览器崩溃 | 重新启动浏览器，从中断处继续（配合 --resume） |
| 计算成本超时 | 判定为合同未签署，回退列表页 |
